ned is a bit like grep and a bit like sed, but not really. FILEs are ascii or
utf-8 text files.

For regex syntax see: http://rust-lang-nursery.github.io/regex/regex/#syntax

Options:
Done!    -p, --pattern PATTERN
                        specify pattern, if the option isn't used the pattern
                        must precede the files, the option allows the pattern
                        to be put after the files for more convenient editing
Done!    -r, --replace REPLACEMENT
                        replace matches, may include named groups
V2    -n, --number N      match/replace N occurrences
V2    -k, --skip N        skip N occurrences before matching/replacing
V2    -b, --backwards     -n --number and -k --skip options count backwards
Done!    -i, --ignore-case   ignore case
Done!    -s, --single        . matches newlines, ^ and $ match beginning and end of
                        each file
Done!    -m, --multiline     multiline, ^ and $ match beginning and end of each
                        line
Done!    -l, --line-oriented
                        operate on lines
Done!    -x, --extended      ignore whitespace and # comments
Done!    -o, --only-matches  show only matches
Done!    -g, --group GROUP   show the match group, specified by number or name
Done!    -v, --no-match      show only non-matching
Done!    -r, --recursive     recurse, follow all symbolic links
Done!    -f, --follow    follow symlinks
Done!        --include FILE_PATTERN
                        search only files that match FILE_PATTERN
Done!        --exclude FILE_PATTERN
                        skip files and directories matching FILE_PATTERN
Done!        --exclude-dir GLOB
                        skip directories matching GLOB
Done!    -c, --colors        show matches in color
Done!        --stdout        output to stdout
Done!    -q, --quiet         suppress all normal output
Done!    -a, --all           do not ignore entries starting with .
Done!    -V, --version       output version information and exit
Done!    -h, --help          print this help menu and exit

Environment:
Done!    NED_DEFAULTS        ned options prepended to the program's arguments

Exit codes:
TODO    0                   matches found/replaced
TODO    1                   no matches

Quiet:
    When -q --quiet is  specified ned tests for matches and returns an exit
    code of 0 if a match is found in ANY file. When -a --all is combined with
    quiet it returns an exit code of 0 if a match is found in ALL files. Quiet
    any matches will only read only as many files as needed to find a match.
    Quiet matches are more performant than non-quiet matches.
